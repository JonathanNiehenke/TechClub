About

  • Released in 1991
  • Clone of Vi
  • Modal interface
    • Normal mode
    • Insertion mode
    • Visual mode
  • Vim was released in 1991 as an improved clone of Vi
  • Vi was created before the adoption of the mouse
  • They use keyboard commands rather than menus, and a modal interface rather than hotkey modifiers.
  • Modal interface changes its functionality depending on the current mode
  • Normal mode is for movement/scrolling, copy/pasting, and entering commands
  • Insertion is for input of text
  • Visual mode is for selecting text
  • And switching modes takes a single keypress much faster than reaching for the mouse.
  • Press the escape key to return to normal mode

Movement

  • Mimic arrow keys: h, j, k, l
  • To word: w, b, e, W, B, E
  • To occurrence: f, t, F, T
  • To boundaries: 0 (zero), ^, $
  • Within Document: gg, G, #g
  • Movement Cheatsheet
  • "h", "j", "k", "l" function like arrow keys but are accessibly on the home row.
  • Think of word movement as "word", "back" and "end"
  • Word movement capitals extend to the whitespace, mimicking the control arrow key hotkeys
  • Occurrence movement "find" moves onto occurrence while "till" moves just before
  • Occurrence movement capitals go backwards
  • Zero moves to the very beginning of the line
  • The carrot moves to the first visible char of the line
  • The dollar sign moves to end of the line.
  • "gg" goes to the document's beginning and "G" to the end
  • A number and "g" move to that line number within the document

Scrolling

  • Automatic
  • Page-up: ctrl+b
  • Page-down: ctrl+f
  • Relative to cursor: zt, zz, zb
  • If the movement is outside of the screen scrolling is automatic
  • I remember page-up and down as backwards and forwards
  • Scrolling relative to the cursor will positions the text about the cursor to that position within the screen
  • So with the cursor positioned about the middle of the screen and we use "zt" the text at the middle shifts up to the top like we were scrolling down

Actions

  • Input: i, I
  • Append: a, A
  • New line: o, O
  • Delete: d, D, dd
  • Change: c, C, cc
  • Yank (Copy): y, yy
  • Paste: p, P
  • Undo & Redo: u, ctrl+r
  • Input: Enters insert mode immediately before the cursor or at the beginning of the line
  • Append: Enters insert mode immediately after the cursor or at the end of the line
  • New line: Enters insert mode on a new line before or after current line
  • Delete: Removes text specified with movement, end of line or entire line
  • Change: Removes text and enters insert mode specified with movement, end of line or entire line
  • Yank: Copies text specified with movement or entire line
  • Paste: Pastes the text last removed or yanked after or before cursor

Search/Replace

  • Search custom pattern: /, ?
  • Search current word: *, #
  • Move to match: n, N
  • Substitute::s/find/replace
  • Search pattern or current word following or preceding the cursor
  • Jump to the next or previous match according to search direction
  • If either substitute command ran as is the first f-i-n-d would get replaced by replace
  • Preceed "s" with "%" to replace within entire file
  • Append "/g" to the substitute command to replace all matches

Custom Search

  • The search patter is in the bottom left corner
  • This pattern include matches within words
  • To highlight the search like in browsers "set hlsearch" command
  • The "noh" command temporarily disables search highlighting

Star Search

  • Search supports regular expressions
  • The star automatically types the search expression matching an entire word

Spell Checking

  • Previous misspell: [s
  • List/Select corrections: z=
