Type Methods

  • Not an operator
  • Like a fastener
  • Enable overloading

" & ".join(["eggs", "bacon", "hash browns"])
# 'eggs & bacon & hash browns'
  • Type methods are not exactly an operator.
  • Type methods may act like a fastener.
  • But they enable operator overloading.
  • In the example.
  • We replace lunch with breakfast.
  • And join a list with ampersand symbols.

Generators

  • Efficient
  • Function
  • Expression
  • Generators - yield individual elements as it goes rather it all at once.
  • Generator function - like a function but a generator:
  • Defined like a function - "def genfunctionName(perameters):"
  • But yields rather than returns - "yield commaSeparatedValues"
  • Called like a function - "functionName(arguments)"
  • Comprehension/generator expressions - highly beneficial syntactical sugar:
  • Expressed with delimiters - (): generator, []: list, {}: set.
  • (returnedExpression for Element in Container if Condition)
  • Dictionary comprehension is expressed like:
  • {keyExpression: valueExpression for value1 in container}
