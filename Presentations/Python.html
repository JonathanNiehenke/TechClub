Programming is the process to building something. So do me the favor of think of something you would like to build. It could be a bird house, a car or whatever and consider how these relate. With that in mind:

  • Physics, the laws of the building environment, relates to programming as syntax.
  • Materials, the things that you build with, relates as data types.
  • Fasteners, what bind the materials together making them useful, relates as operators.
  • Logic, decision making, relates to flow control.
  • Blueprints, the instructions to repetitious tasks, relates as functions and classes.

What is Python

  • Interpreted
  • Dynamically typed
  • Multi-paradigm
  • By Guido van Rossum
  • Released in 1991
  • "Python is an interpreted, dynamically typed, multi-paradigm programming language written by Guido van Rossum and first released in February 1991."
  • Supported and developed through the Python Software Foundation.
  • Interpreted means it is executed by a program rather than by the system directly
  • *Like Java (another programming language), Python compiles the source code before interpretation*
  • Multi-paradigm means it supports many styles of programming.
  • Dynamically typed means the variable's data type is inferred rather than declared.

Generators

  • Efficient
  • Function
  • Expression
  • Generators - yield individual elements as it goes rather it all at once.
  • Generator function - like a function but a generator:
  • Defined like a function - "def genfunctionName(perameters):"
  • But yields rather than returns - "yield commaSeparatedValues"
  • Called like a function - "functionName(arguments)"
  • Comprehension/generator expressions - highly beneficial syntactical sugar:
  • Expressed with delimiters - (): generator, []: list, {}: set.
  • (returnedExpression for Element in Container if Condition)
  • Dictionary comprehension is expressed like:
  • {keyExpression: valueExpression for value1 in container}
